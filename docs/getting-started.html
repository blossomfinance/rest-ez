linkInstallation

Install with npm:

1link$npm install just-api

!!! note To run just-api, you will need Node.js v7.10.0 or newer.

linkGetting Started

1link$mkdir specs

1link$vim specs/starwars_service.yml

Enter below specification in YAML file

1linkmeta:

2link  name: "Star Wars suite"

3linkconfiguration:

4link  scheme: "https"

5link  host: "swapi.co"

6link  base_path: "/api"

7linkspecs:

8link  - name: "get Luke Skywalker info"

9link    request:

10link      path: "/people/1/"

11link      method: "get"

12link    response:

13link      status_code: 200

14link      headers:

15link        - name: "content-type"

16link          value: !!js/regexp application/json      

17link      json_data:

18link        - path: "$.name"

19link          value: "Luke Skywalker"

linkTesting GraphQL APIs

Following example tests a GraphQL API that returns location for a given ip address.

Create a YAML suite and run just-api.

1linkmeta:

2link  name: GraphQL location service

3linkconfiguration:

4link  host: api.graphloc.com

5link  scheme: https

6linkspecs:

7link  - name: Get Location of a given ip address

8link    request:

9link      method: post

10link      path: /graphql

11link      headers:

12link        - name: content-type

13link          value: application/json

14link      payload:

15link        body:

16link          type: json

17link          content:

18link            query: >

19link                   {

20link                    getLocation(ip: "8.8.8.8") {

21link                      country {

22link                        iso_code

23link                      }

24link                     }

25link                    }

26link            variables: null

27link            operationName: null

28link    response:

29link      status_code: 200

30link      json_data:

31link        - path: $.data.getLocation.country.iso_code

32link          value: US

linkA chained request flow with hook and custom validation

When you need to test complex chained API flows, run dependencies in hooks to fetch pre-requisite data and pass it to actual test.

Following example shows how to run dependencies using a hook, get data and validating response with a custom validator function.

1linkmeta:

2link  name: Starwars suite

3linkconfiguration:

4link  scheme: https

5link  host: swapi.co

6link  base_path: /api

7linkspecs:

8link  - name: get R2-D2 info

9link    request:

10link      path: /people/3/

11link      method: get

12link    response:

13link      status_code: 200

14link      json_data:

15link        - path: $.name

16link          value: R2-D2

17link

18link  - name: search R2-D2 info

19link    before_test:

20link      run_type: inline

21link      inline:

22link        function: !js/asyncFunction >

23link          async function() {

24link            var response = await this.runSpec('get R2-D2 info');

25link            var jsonData = JSON.parse(response.body);

26link            this.test.query_params = { name:  jsonData.name };

27link          }

28link    request:

29link      path: /people

30link      method: get

31link    response:

32link      status_code: 200

33link      custom_validator:

34link        run_type: inline

35link        inline:

36link          function: !!js/function >

37link            function() {

38link              var jsonData = JSON.parse(this.response.body);

39link              var r2d2 = jsonData.results.find(result => result.name === 'R2-D2');

40link

41link              if (!r2d2) throw new Error('R2-D2 not returned in search results');

42link            }

!!! note You can also place custom JS functions in a module and specify the function name and path to module in YAML.
